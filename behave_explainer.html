① Creative Extraction (LLM Pre-Pass)

User only provides brand name + ad image. Claude Sonnet sees the image and extracts: category (from 18 canonical B2C categories), headline, CTA, brand positioning, era, marketing tone, festival context, visual style, pricing signals. No manual form-filling.

② Cohort Selection

Rule-based matching: extracted category → persona affinities → rank by affinity score. Top N selected + edge cases. Or manual pick from 20 personas. Memory scope controls which past exposures each persona recalls.

③ Vision-Native Persona Evaluation

Each persona receives: their full soul profile + raw ad image + CreativeCard metadata + accumulated memories. Outputs: 5 dimensional scores (attention, relevance, trust, desire, clarity), primary action, verbatim in their voice, key themes.

④ Critic → Score → Recommend

Critic agent checks each eval for sycophancy, persona consistency, cultural authenticity. Passed evals aggregate into a scorecard. Recommendation agent decides: SCALE / ITERATE / KILL + edit playbook.

04

Databricks Pipeline

Medallion Architecture — Bronze → Silver → Gold

Bronze — Raw

  • persona_seed (20 JSONs)
  • creative_uploads
  • evaluations (raw LLM output)
  • critic_verdicts
  • memory_nodes
  • run_metadata

Silver — Validated

  • personas (schema-checked)
  • creative_cards (normalized)
  • evaluations (scores in range, reasoning ≥10 chars)
  • memory_nodes (node_id valid)
  • quarantine + rejection reasons

Gold — Analytics

  • creative_scorecards (aggregated)
  • segment_summaries
  • run_audit_log
  • MLflow experiments

Data Quality Controls

DLT-style expectations at silver layer: required fields, score ranges, duplicate detection, category vocabulary enforcement. Failed records → quarantine table with rejection reasons. Critic Agent adds a second quality gate: persona consistency, sycophancy detection, cultural authenticity checks.

Pipeline Execution

Triggered micro-batch, not always-on streaming. Each simulation auto-ingests: raw JSONs → DBFS → 04_simulation_ingest.py processes bronze → silver → gold. Deliberate design: creative testing is request-driven, not continuous. MLflow logs every run (score, cost, tokens, pass rate).

05

Agent Architecture

Three-layer agents with accumulating memory

Layer 1: The Soul

Deep persona profile: purchase psychology, price sensitivity, brand trust formation, social proof dependency, cultural context (language, festivals, family, city tier), category involvement. Not just demographics.

Layer 2: The Memory

Append-only memory stream: observations, preferences, beliefs, reflections. Full-dump mode (200K context). Reflections trigger at cumulative importance > 150. Memory scoped by category/brand/full. Personas evolve across runs.

Layer 3: The Action

SCROLL_PAST → PAUSE → READ → ENGAGE → CLICK → SAVE → SHARE → PURCHASE_INTENT → NEGATIVE_REACT. One primary action + 5 dimensional scores + reasoning + verbatim in persona's voice.

Critic Agent (Quality Gate)

Scores 4 dimensions (0-10): persona consistency, sycophancy, cultural authenticity, action-reasoning alignment. FAIL → quarantine. Not summarization — active data quality enforcement.

Recommendation Agent (Business Action)

Consumes gold scorecard + segment summaries + top verbatims. Outputs: SCALE / ITERATE / KILL / SPLIT decision + confidence score + prioritized edit playbook with specific element-level changes.

Tech Stack & Design Decisions

Why we built it this way

Pydantic v2 Data Models

Structured output enforcement at API level. Every LLM response validated against schema. No parsing errors in production.

Anthropic Claude Sonnet 4 LLM Engine

Vision-native: personas see the actual ad image, not just extracted text. 200K context window enables full memory dump mode.

Delta Lake + Unity Catalog Storage

ACID transactions for evaluation writes. Time-travel for cross-run comparison. 3-level namespace: catalog.schema.table. Governed lineage.

MLflow Tracking

Every simulation run = an MLflow experiment. Logged: overall score, cost, token usage, critic pass rate, memory nodes created. Enables cross-run comparison.

FastAPI + Vanilla JS Dashboard

Zero build tools. Async endpoints. Live simulation with progress feedback. Pre-loaded Zepto case study for instant demo. Password-gated access.

Triggered Micro-Batch Execution

Not always-on streaming — deliberate choice. Creative testing is request-driven. Each simulation triggers ingest → transform → aggregate pipeline. Cheaper, simpler, correct.
